# 🚀 OVH Phantom Sniper - 配置说明（一页纸）

## 📋 .env 文件需要配置什么？

### ✅ 必须配置（只有 1 项）

```env
API_SECRET_KEY=<你的密钥>
```

**生成密钥命令：**
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

---

### 🔧 可选配置

```env
PORT=5000                      # 后端端口（可不改）
DEBUG=false                    # 调试模式（生产环境用false）
ENABLE_API_KEY_AUTH=true       # 启用密钥验证（建议保持true）
```

---

## 📄 完整的 .env 文件内容

```env
# API 安全密钥（必须修改）
API_SECRET_KEY=ovh-phantom-sniper-2024-secret-key

# 应用配置
PORT=5000
DEBUG=false
ENABLE_API_KEY_AUTH=true
```

---

## ⚡ 快速开始

### 1. 创建配置文件

```bash
cd backend
cp env_example.txt .env
```

### 2. 生成密钥

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

复制输出的密钥

### 3. 编辑 .env

用记事本或任何编辑器打开 `backend/.env`，将 `API_SECRET_KEY` 的值改为刚才生成的密钥

### 4. 启动

```bash
# 后端
cd backend
pip install -r requirements.txt
python app.py

# 前端（新窗口）
npm run dev
```

### 5. 浏览器配置

1. 访问 `http://localhost:5173/settings`
2. 在 **"🔐 网站安全密钥"** 处输入与 .env 相同的密钥
3. 填写 OVH API 凭据
4. 保存

---

## ❓ 其他配置在哪里？

**不需要在 .env 中配置！**

以下内容在**浏览器设置页面**配置：
- OVH API 凭据（APP_KEY, APP_SECRET, CONSUMER_KEY）
- Telegram 通知（TG_TOKEN, TG_CHAT_ID）
- 区域设置（ZONE, IAM）

---

## 🔒 重要提示

⚠️ **API_SECRET_KEY 的值必须在两处保持一致：**

1. 后端 `.env` 文件
2. 前端设置页面

如果不一致，API 请求会失败！
